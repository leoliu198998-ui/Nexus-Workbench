# Specification - Layout Optimization & Dashboard

## 1.0 概述
随着项目功能的扩展，现有的单一功能页面（Excel 导出向导）已不足以承载整个平台。本 Track 旨在重构前端布局，引入一个仪表盘页面作为主入口，并为未来的工具扩展提供统一的导航和卡片式入口。

## 2.0 需求细节

### 2.1 路径重构 (Routing)
- **主页 (`/`)**: 变更为“仪表盘 (Dashboard)”，展示所有可用工具的卡片列表。
- **Excel 导出向导 (`/apps/excel-export`)**: 原有的主页逻辑整体迁移至此路径。

### 2.2 仪表盘 UI (Dashboard UI)
- **布局**: 采用网格布局 (Grid) 展示工具卡片。
- **卡片 (Tool Card)**: 每个工具入口以卡片形式呈现，包含：
  - **名称**: 清晰的工具标题。
  - **描述**: 简洁的功能说明。
  - **图标**: 与工具功能相关的可视化标识。
  - **状态标签**: 显示工具的当前状态（如：Active, Beta, Stable）。

### 2.3 导航增强 (Navigation)
- 引入一个简单的头部导航栏 (Header) 或面包屑 (Breadcrumbs)，允许用户从工具页面快速返回仪表盘。

## 3.0 技术实现
- **前端**: 使用 Next.js 的文件系统路由进行页面迁移。
- **组件**: 使用 shadcn/ui 的 Card 组件构建仪表盘卡片。

## 4.0 验收标准 (Acceptance Criteria)
1. 访问 `http://localhost:3000` 看到的是仪表盘工具列表。
2. 访问 `http://localhost:3000/apps/excel-export` 能够正常使用原有的向导功能。
3. 仪表盘卡片信息完整（名称、描述、图标、标签）。
4. 页面导航逻辑清晰，支持从向导页面返回主页。

## 5.0 不在范围内 (Out of Scope)
- 增加除了 Excel 导出之外的其他实际功能工具。
- 实现复杂的多级权限或用户个性化仪表盘。
